How do I use the Common module settings section of an activity or resource?

You will learn how to use the Common module settings in your course in TITANium.

The Common module settings is a section in each Resource and Activity within your course in TITANium. Follow these steps to learn how to adjust those settings.

Resources have fewer settings than do Activities.

Do you want to learn about the Common module settings in a(n):

Activity?

Follow theses steps to adjust the settings in an Activity (such as an Assignment, Chat, Choice, Forum, Database, etc.).

Visible

The Visible field by default is set for Show. If you want to hide this activity select Hide.

Note: Keep in mind that hiding this activity will also hide it in the Gradebook (if it has a grade value).

ID number

The ID number field is only needed when you are using a custom formula in your gradebook. Read this guide for an example use of a custom calcluation in your gradebook.

Group mode

Three are three choices for Group mode: No groups, Separate groups, and Visible groups.

No groups means that all students in the course can access the activity. So, for example, when you create a forum in which the entire class should be able to read and make posts.

Separate groups is for an activity where students should only see and interact with members of their own group, such as for a group forum or group assignment (where one of the group members submits the assignment for the entire group, for example).

Visible groups is for an activity where you want student to interact with their own group yet still be able to see what other groups are doing. This can be used in a Wiki, for example, where each group creates a wiki on a particular topic. Plus, all students can see each other's wiki page(s).

If Group mode is greyed out, then your Course settings have been set to force either Visible groups or Separate groups for all activities.

Grouping

Grouping is only enabled if either Separate groups or Visible groups is selected for Group mode. The default for Grouping is None. A grouping is not required for using Separate or Visible groups.

The purpose of using grouping is when you want to have students in more than one group. For example, maybe you have students in a group for a particular forum or a wiki, but want them in a different group for a final presentation. Groupings is how sets of groups are managed.

Read this guide on creating and using groups and groupings.

Add group/grouping access restriction.

Once Visible groups or Separate groups has been selected, you can then click on the Add group/grouping access restriction button. This will create a Restrict access setting prohibiting non-group or non-grouping members from clicking on the activity.

A grouping restriction is now visible in the Restrict access section just below.

Since a Grouping was selected, the restriction is based on that grouping. Therefore, to access this activity, a student must belong to a group within the "Renaissance Master" grouping. Click on the X to the right to remove this restriction if desired.

For more information about using Restrict access, read this guide.

Resource?

Follow these steps to learn how to adjust the settings in a Resource (such as a Page, Book, Label, File, etc.).

Note that the Common module settings for a Resource usually only has two fields: Visible and ID number.

Visible

The Visible field by default is set for Show. If you want to hide this resource, select Hide.

ID number

The ID number field is only needed when you are using a custom formula in your gradebook. Read this guide for an example use of a custom calculation in your gradebook.

Our current version of TITANium 3.2 does not utilize the ID number field in Resources. Only Activities reporting to the gradebook have a use for the ID number field.
